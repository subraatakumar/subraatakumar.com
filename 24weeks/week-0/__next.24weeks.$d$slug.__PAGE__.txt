<p class="d180-md-p"><em>Focus: Mastering the interface between traditional code and LLMs.</em></p>
<ul class="d180-md-ul">
<li class="d180-md-li"><strong>Week 1: The Multi-Model Evaluator.</strong> Build a side-by-side comparison tool for GPT-4, Claude, and Gemini to test architectural logic.</li>
<li class="d180-md-li"><strong>Week 2: Local RAG (Retrieval-Augmented Generation).</strong> Create a tool to "chat" with your own local Markdown or PDF notes using a Vector DB (ChromaDB).</li>
<li class="d180-md-li"><strong>Week 3: Structured Data Factory.</strong> A service that converts messy text (emails/receipts) into strict, validated JSON using Pydantic.</li>
<li class="d180-md-li"><strong>Week 4: The Voice-to-Jira Pipeline.</strong> Use Whisper (audio-to-text) to turn voice memos into formatted project tickets automatically.</li>
<li class="d180-md-li"><strong>Week 5: The LLM-as-a-Judge.</strong> Build an automated testing script where a "stronger" model audits the code quality of a "smaller" model.</li>
<li class="d180-md-li"><strong>Week 6: Multimodal UI Auditor.</strong> An app that "sees" a website screenshot and generates a report on UI/UX flaws.</li>
<li class="d180-md-li"><strong>Week 7: Token &#x26; Cost Dashboard.</strong> A middleware that tracks API usage and predicts monthly spend based on current prompt volume.</li>
<li class="d180-md-li"><strong>Week 8: Phase 1 Portfolio Site.</strong> Deploy a centralized dashboard hosting all previous 7 micro-tools.</li>
</ul>

<h2 id="phase-2-autonomous-agents-x26-orchestration-weeks-916" class="d180-md-h2"><strong>Phase 2: Autonomous Agents &#x26; Orchestration (Weeks 9–16)</strong></h2>
<p class="d180-md-p"><em>Focus: Moving from "Chat" to "Task Execution" using Agentic workflows.</em></p>
<ul class="d180-md-ul">
<li class="d180-md-li"><strong>Week 9: The Autonomous Research Agent.</strong> An agent that browses 5+ web sources and writes a technical briefing on a specific 2030 trend.</li>
<li class="d180-md-li"><strong>Week 10: Natural Language SQL Agent.</strong> Build a bridge where you ask "Who are my top customers?" and the agent writes and runs the SQL.</li>
<li class="d180-md-li"><strong>Week 11: The Multi-Agent Debate.</strong> Set up two agents—a "Developer" and a "Security Specialist"—to argue over a code snippet until it’s perfect.</li>
<li class="d180-md-li"><strong>Week 12: Long-Term Memory (Graph RAG).</strong> Use a Graph Database (Neo4j) to help an agent remember relationships between concepts over time.</li>
<li class="d180-md-li"><strong>Week 13: Automated PR Reviewer.</strong> A GitHub bot that automatically reviews incoming code for performance bottlenecks.</li>
<li class="d180-md-li"><strong>Week 14: Human-in-the-Loop Workflow.</strong> An agent that researches and drafts emails but "pauses" for your physical button-click approval before sending.</li>
<li class="d180-md-li"><strong>Week 15: Local Model Edge Deployment.</strong> Run a quantized Llama-3 model locally using Ollama to prove "Privacy-First" AI capabilities.</li>
<li class="d180-md-li"><strong>Week 16: The Autonomous SDR.</strong> An agent that finds potential leads on LinkedIn and drafts hyper-personalized outreach based on their recent posts.</li>
</ul>

<h2 id="phase-3-production-scale-x26-safety-weeks-1724" class="d180-md-h2"><strong>Phase 3: Production, Scale &#x26; Safety (Weeks 17–24)</strong></h2>
<p class="d180-md-p"><em>Focus: Professionalizing the stack for enterprise-grade reliability.</em></p>
<ul class="d180-md-ul">
<li class="d180-md-li"><strong>Week 17: The LLM Firewall.</strong> Build a security layer that detects and sanitizes "Prompt Injection" attempts in real-time.</li>
<li class="d180-md-li"><strong>Week 18: Vector DB Migration Logic.</strong> Write a script to migrate high-dimensional data between two different Vector providers without downtime.</li>
<li class="d180-md-li"><strong>Week 19: Targeted Fine-Tuning.</strong> Fine-tune a small model (Mistral/Phi) on a specific niche dataset (e.g., medical or legal) using LoRA.</li>
<li class="d180-md-li"><strong>Week 20: Serverless Inference Scaling.</strong> Deploy your AI logic to AWS Lambda or Vercel Functions and benchmark cold-start latency.</li>

<li class="d180-md-li"><strong>Week 23: The Compliance Auditor.</strong> A tool that scans AI outputs for PII (Personally Identifiable Information) before it hits the UI.</li>
<li class="d180-md-li"><strong>Week 24: The 2030 Capstone.</strong> Combine the best elements of the previous 23 weeks into one "AI-Native" SaaS product.</li>
