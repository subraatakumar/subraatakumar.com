a:T5355,<h2 id="week-2-mission-build-a-local-rag-engine-that-you-can-trust" class="d180-md-h2"><strong>Week 2 Mission: Build a Local RAG Engine That You Can Trust</strong></h2>
<p class="d180-md-p">Week 1 proved we can compare model outputs.</p>
<p class="d180-md-p">Week 2 is about <strong>grounding</strong> those outputs in your own documents.</p>
<p class="d180-md-p">Goal: Build a Local RAG system where answers come from your files (Markdown/PDF), with visible citations and deterministic architecture.</p>
<hr class="d180-md-hr" />
<h2 id="what-this-plan-covers" class="d180-md-h2"><strong>What This Plan Covers</strong></h2>
<p class="d180-md-p">This architecture plan includes both:</p>
<ul class="d180-md-ul">
<li class="d180-md-li"><strong>Option A: LM Studio Local RAG Architecture</strong></li>
<li class="d180-md-li"><strong>Option B: Microsoft Foundry Local RAG Architecture</strong></li>
</ul>
<p class="d180-md-p">And then locks implementation choice:</p>
<ul class="d180-md-ul">
<li class="d180-md-li"><strong>Final build path for this project: Microsoft Foundry Local RAG</strong></li>

<h2 id="decision-summary-locked" class="d180-md-h2"><strong>Decision Summary (Locked)</strong></h2>
<h3 id="why-compare-both" class="d180-md-h3">Why compare both?</h3>
<p class="d180-md-p">Both are local-first inference approaches, but they differ in ecosystem and operational model.</p>
<h3 id="final-decision-for-week-2-build" class="d180-md-h3">Final decision for Week 2 build</h3>
<p class="d180-md-p">We will implement with <strong>Microsoft Foundry Local</strong> for this project, because:</p>
<ol class="d180-md-ol">
<li class="d180-md-li">It aligns with your current direction and tooling interest.</li>
<li class="d180-md-li">It gives a clear local runtime contract (CLI/SDK/REST style).</li>
<li class="d180-md-li">It keeps a clean path to Microsoft ecosystem integration later (if needed).</li>
</ol>
<p class="d180-md-p">LM Studio architecture is still documented fully as a fallback/alternative.</p>
<hr class="d180-md-hr" />
<h2 id="shared-week-2-architecture-applies-to-both-options" class="d180-md-h2"><strong>Shared Week-2 Architecture (Applies to Both Options)</strong></h2>
<p class="d180-md-p">Regardless of runtime, the RAG pipeline is the same:</p>
<ol class="d180-md-ol">
<li class="d180-md-li"><strong>Ingest files</strong> (<code class="d180-md-inline-code">.md</code>, <code class="d180-md-inline-code">.pdf</code>)</li>
<li class="d180-md-li"><strong>Extract text</strong> and normalize content</li>
<li class="d180-md-li"><strong>Chunk text</strong> into retrievable units</li>
<li class="d180-md-li"><strong>Generate embeddings</strong> for each chunk</li>
<li class="d180-md-li"><strong>Store vectors</strong> in Chroma with metadata</li>
<li class="d180-md-li"><strong>Retrieve top-k chunks</strong> for user query</li>
<li class="d180-md-li"><strong>Generate answer</strong> from retrieved context</li>
<li class="d180-md-li"><strong>Return citations</strong> mapped to chunk metadata</li>
</ol>

<h2 id="chunking-strategy-locked" class="d180-md-h2"><strong>Chunking Strategy (Locked)</strong></h2>
<ul class="d180-md-ul">
<li class="d180-md-li"><strong>Chunk size:</strong> 800 characters</li>
<li class="d180-md-li"><strong>Overlap:</strong> 120 characters</li>
<li class="d180-md-li"><strong>Splitter rule:</strong> paragraph-aware first, fallback to hard-length split</li>
<li class="d180-md-li"><strong>Normalization:</strong> collapse repeated spaces/newlines, preserve bullet and heading boundaries where possible</li>
</ul>
<p class="d180-md-p">Why this default:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">Keeps chunks small enough for precise retrieval</li>
<li class="d180-md-li">Preserves enough context continuity through overlap</li>
<li class="d180-md-li">Works well for both markdown notes and extracted PDF text</li>
</ul>

<li class="d180-md-li"><strong>Unsupported mime type</strong>
<ul class="d180-md-ul">
<li class="d180-md-li">Skip with explicit reason in <code class="d180-md-inline-code">skipped[]</code></li>
</ul>
</li>
<li class="d180-md-li"><strong>No extractable text</strong>
<ul class="d180-md-ul">
<li class="d180-md-li">Skip and report <code class="d180-md-inline-code">empty_text</code></li>
</ul>
</li>
</ol>
<h3 id="retrievalchat-failures" class="d180-md-h3">Retrieval/chat failures</h3>
<ol class="d180-md-ol">
<li class="d180-md-li"><strong>Vector DB empty</strong>
<ul class="d180-md-ul">
<li class="d180-md-li">Return graceful message: "No indexed documents found. Ingest files first."</li>
</ul>
</li>
<li class="d180-md-li"><strong>No relevant chunks</strong>
<ul class="d180-md-ul">
<li class="d180-md-li">Return low-confidence answer + warning + empty citations</li>
</ul>
</li>
<li class="d180-md-li"><strong>Local model unavailable</strong>
<ul class="d180-md-ul">
<li class="d180-md-li">Return actionable error: runtime not running/model not loaded</li>
</ul>
</li>
<li class="d180-md-li"><strong>Timeout during generation</strong>
<ul class="d180-md-ul">
<li class="d180-md-li">Return partial failure with retry suggestion</li>
</ul>
</li>
</ol>
<hr class="d180-md-hr" />
<h2 id="option-a-lm-studio-local-rag-architecture" class="d180-md-h2"><strong>Option A: LM Studio Local RAG Architecture</strong></h2>
<h3 id="runtime" class="d180-md-h3">Runtime</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">Local generation model hosted via LM Studio local server endpoint</li>
<li class="d180-md-li">Separate embedding model via LM Studio (if available in your setup) or fallback embedding runtime</li>
</ul>
<h3 id="service-wiring" class="d180-md-h3">Service wiring</h3>
<ul class="d180-md-ul">
<li class="d180-md-li"><code class="d180-md-inline-code">Ingest Service</code> -> parser -> chunker -> embedder -> Chroma</li>
<li class="d180-md-li"><code class="d180-md-inline-code">Chat Service</code> -> query embed -> retrieve top-k -> prompt builder -> LM Studio generation</li>
</ul>
<h3 id="pros" class="d180-md-h3">Pros</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">Quick start with user-friendly desktop workflow</li>
<li class="d180-md-li">Easy model switching in UI</li>
<li class="d180-md-li">Great for experimentation and demos</li>
</ul>
<h3 id="tradeoffs" class="d180-md-h3">Tradeoffs</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">Environment parity across machines can vary</li>
<li class="d180-md-li">Embedding model/runtime setup may be less standardized depending on local stack</li>
</ul>
<h3 id="recommended-when" class="d180-md-h3">Recommended when</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">Fast prototyping and model experimentation is top priority</li>
</ul>
<hr class="d180-md-hr" />
<h2 id="option-b-microsoft-foundry-local-rag-architecture-selected" class="d180-md-h2"><strong>Option B: Microsoft Foundry Local RAG Architecture (Selected)</strong></h2>
<h3 id="runtime" class="d180-md-h3">Runtime</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">Generation model served by Foundry Local runtime</li>
<li class="d180-md-li">Embedding model also served via Foundry Local where supported by selected model set</li>
</ul>
<h3 id="service-wiring" class="d180-md-h3">Service wiring</h3>
<ul class="d180-md-ul">
<li class="d180-md-li"><code class="d180-md-inline-code">Ingest API</code> calls Foundry Local embedding endpoint for each chunk</li>
<li class="d180-md-li">Chroma stores vectors and metadata</li>
<li class="d180-md-li"><code class="d180-md-inline-code">Chat API</code> retrieves top-k chunks and calls Foundry Local generation endpoint with contextual prompt</li>
</ul>
<h3 id="pros" class="d180-md-h3">Pros</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">More standardized local runtime contract for enterprise-like workflows</li>
<li class="d180-md-li">Clear alignment with Microsoft AI platform path</li>
<li class="d180-md-li">Better long-term fit for structured local deployment discipline</li>
</ul>
<h3 id="tradeoffs" class="d180-md-h3">Tradeoffs</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">Model availability varies by platform/runtime support</li>
<li class="d180-md-li">Initial local setup can be more procedural than LM Studio</li>
</ul>
<h3 id="recommended-when" class="d180-md-h3">Recommended when</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">You want disciplined local architecture with Microsoft ecosystem alignment</li>
</ul>
<hr class="d180-md-hr" />
<h2 id="selected-build-blueprint-microsoft-foundry-local" class="d180-md-h2"><strong>Selected Build Blueprint (Microsoft Foundry Local)</strong></h2>
<h3 id="components" class="d180-md-h3">Components</h3>
<ol class="d180-md-ol">
<li class="d180-md-li"><strong>Frontend UI</strong> (Next.js page)
<ul class="d180-md-ul">
<li class="d180-md-li">Upload documents</li>
<li class="d180-md-li">Ask grounded questions</li>
<li class="d180-md-li">Render citations and retrieval info</li>
</ul>
</li>
<li class="d180-md-li"><strong>Ingest API</strong> (<code class="d180-md-inline-code">POST /api/ingest</code>)
<ul class="d180-md-ul">
<li class="d180-md-li">Parse + chunk + embed + upsert to Chroma</li>
</ul>
</li>
<li class="d180-md-li"><strong>Chat API</strong> (<code class="d180-md-inline-code">POST /api/chat</code>)
<ul class="d180-md-ul">
<li class="d180-md-li">Embed query + retrieve + generate + cite</li>
</ul>
</li>
<li class="d180-md-li"><strong>Vector Store</strong> (Chroma)
<ul class="d180-md-ul">
<li class="d180-md-li">Local persistent collection</li>
</ul>
</li>
<li class="d180-md-li"><strong>Foundry Local Runtime</strong>
<ul class="d180-md-ul">
<li class="d180-md-li">One generation model + one embedding model</li>
</ul>
</li>
</ol>
<h3 id="default-operational-values" class="d180-md-h3">Default operational values</h3>
<ul class="d180-md-ul">
<li class="d180-md-li"><code class="d180-md-inline-code">chunkSize = 800</code></li>
<li class="d180-md-li"><code class="d180-md-inline-code">chunkOverlap = 120</code></li>
<li class="d180-md-li"><code class="d180-md-inline-code">topK = 5</code></li>
<li class="d180-md-li"><code class="d180-md-inline-code">temperature = 0.2</code></li>
</ul>
<h3 id="security-baseline" class="d180-md-h3">Security baseline</h3>
<ul class="d180-md-ul">
<li class="d180-md-li">Local-only storage by default</li>
<li class="d180-md-li">No external doc upload by default</li>
<li class="d180-md-li">No secret values written to client logs</li>
</ul>
