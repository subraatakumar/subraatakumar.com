<h2 id="the-why-why-local-rag-matters" class="d180-md-h2"><strong>The Why: Why Local RAG Matters</strong></h2>
<p class="d180-md-p">LLMs are powerful, but they are not connected to your private, latest documents by default.</p>
<p class="d180-md-p">RAG solves this by retrieving relevant chunks from your corpus and injecting them into the prompt at answer time.</p>
<p class="d180-md-p">Benefits:</p>
<ul class="d180-md-ul">
<li class="d180-md-li"><strong>Lower hallucination risk:</strong> answers are anchored to retrieved context.</li>
<li class="d180-md-li"><strong>Fresh knowledge:</strong> you can query documents created after model training.</li>
<li class="d180-md-li"><strong>Privacy:</strong> files can stay local when ingestion, retrieval, and generation run on your machine.</li>
</ul>
<hr class="d180-md-hr" />
<h2 id="system-overview" class="d180-md-h2"><strong>System Overview</strong></h2>
<p class="d180-md-p">The architecture has two loops: <strong>Ingestion</strong> and <strong>Retrieval/Chat</strong>.</p>
<p class="d180-md-p"><img src="/24weeks/week2/local_rag.png" class="rounded-md shadow-sm mx-auto block" alt="Local RAG architecture overview"></p>
<h3 id="phase-1-ingestion" class="d180-md-h3"><strong>Phase 1: Ingestion</strong></h3>
<ol class="d180-md-ol">
<li class="d180-md-li">User uploads <code class="d180-md-inline-code">.pdf</code>, <code class="d180-md-inline-code">.md</code>, or <code class="d180-md-inline-code">.txt</code>.</li>
<li class="d180-md-li">Frontend sends content to <code class="d180-md-inline-code">POST /api/ingest</code>.</li>
<li class="d180-md-li">Backend extracts text and applies chunking.</li>
<li class="d180-md-li">Chunks are embedded using <code class="d180-md-inline-code">nomic-embed-text-v1</code> via Foundry Local.</li>
<li class="d180-md-li">Chunk text + vectors + metadata are stored in Chroma.</li>
</ol>
<h3 id="phase-2-retrieval-answer" class="d180-md-h3"><strong>Phase 2: Retrieval + Answer</strong></h3>
<ol class="d180-md-ol">
<li class="d180-md-li">User submits a question.</li>
<li class="d180-md-li">Frontend calls <code class="d180-md-inline-code">POST /api/chat</code>.</li>
<li class="d180-md-li">Query is embedded with the same embedding model.</li>
<li class="d180-md-li">Chroma runs vector similarity search (<code class="d180-md-inline-code">topK</code>, optional threshold).</li>
<li class="d180-md-li">Retrieved chunks are passed to local generation runtime.</li>
<li class="d180-md-li">API returns grounded answer + citations.</li>
<li class="d180-md-li">If generation fails, API falls back to retrieved excerpts.</li>
</ol>
<hr class="d180-md-hr" />
<h2 id="architecture-contract-week-2-locked-defaults" class="d180-md-h2"><strong>Architecture Contract (Week 2 Locked Defaults)</strong></h2>
<h3 id="chunking-defaults" class="d180-md-h3"><strong>Chunking Defaults</strong></h3>
<ul class="d180-md-ul">
<li class="d180-md-li"><code class="d180-md-inline-code">chunkSize = 800</code></li>
<li class="d180-md-li"><code class="d180-md-inline-code">chunkOverlap = 120</code></li>
<li class="d180-md-li">paragraph-aware split first, hard split fallback</li>
</ul>
<h3 id="retrieval-defaults" class="d180-md-h3"><strong>Retrieval Defaults</strong></h3>
<ul class="d180-md-ul">
<li class="d180-md-li"><code class="d180-md-inline-code">topK = 5</code></li>
<li class="d180-md-li"><code class="d180-md-inline-code">minScore = 0.2</code></li>
<li class="d180-md-li">return best available context with warning if strict threshold gives low matches</li>
</ul>

export async function ingestDocument(args: {
  source: string;
  text: string;
  chunkSize: number;
  chunkOverlap: number;
}) {
  const parts = chunkText(args.text, args.chunkSize, args.chunkOverlap);
  if (parts.length === 0) return [];

  const embeddings = await getEmbeddings(parts);
  const collection = await getCollection();

  const records = parts.map((part, index) => {
    const chunkId = `${args.source}::${index}`;
    return {
      id: chunkId,
      chunk_id: chunkId,
      source: args.source,
      page: null,
      text: part,
      created_at: new Date().toISOString(),
    };
  });

  await collection.add({
    ids: records.map((r) => r.id),
    embeddings,
    documents: records.map((r) => r.text),
    metadatas: records.map((r) => ({
      source: r.source,
      page: r.page,
      chunk_id: r.chunk_id,
      created_at: r.created_at,
    })),
  });

  return records;
}
