        <pre class="d180-code-pre line-numbers"><code class="language-tsx">import { getRealm } from '../../core/storage/realm';

export type EntryInput = {
  id: string;
  title: string;
  content: string;
  createdAt: Date;
};

export class ArchiveRepository {

  create(entry: EntryInput) {
    const realm = getRealm();

    realm.write(() => {
      realm.create('Entry', entry);
    });
  }

  getAll() {
    const realm = getRealm();
    return realm.objects('Entry').sorted('createdAt', true);
  }

  delete(id: string) {
    const realm = getRealm();

    realm.write(() => {
      const entry = realm.objectForPrimaryKey('Entry', id);
      if (entry) {
        realm.delete(entry);
      }
    });
  }
}
</code></pre>

<li class="d180-md-li">No timeline screen</li>
<li class="d180-md-li">No tags</li>
<li class="d180-md-li">No search</li>
</ul>
<p class="d180-md-p">We are respecting boundaries.</p>
<hr class="d180-md-hr" />
<h2 id="67-what-this-phase-clarifies" class="d180-md-h2">6.7 What This Phase Clarifies</h2>
<p class="d180-md-p">Repository:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">Abstracts database access.</li>
<li class="d180-md-li">Protects storage structure.</li>
<li class="d180-md-li">Prevents cross-layer contamination.</li>
<li class="d180-md-li">Makes future database swap possible.</li>
<li class="d180-md-li">Keeps modules clean.</li>
</ul>
<p class="d180-md-p">Encryption protects data.</p>
<p class="d180-md-p">Repository protects access.</p>
<p class="d180-md-p">Both are infrastructure decisions.</p>
<hr class="d180-md-hr" />
<h1 id="-day-6-outcome" class="d180-md-h1">🎯 Day 6 Outcome</h1>
<p class="d180-md-p">Encrypted storage + structured data access.</p>
<p class="d180-md-p">Now the system can persist entries safely.</p>
<p class="d180-md-p">Tomorrow (Day 7):</p>
<p class="d180-md-p">Service layer introduces business rules.</p>
<p class="d180-md-p">Warehouse manager enters the system.</p>
<hr class="d180-md-hr" />
<p class="d180-md-p">If you’re ready, say:</p>
<blockquote class="d180-md-blockquote">
<p class="d180-md-p">Repository layer stable.</p>
</blockquote>
<p class="d180-md-p">And we move to Day 7 — Service implementation with real validation logic.</p>

<h2 id="-day-6-practical-exploration" class="d180-md-h2">🔧 Day 6 — Practical Exploration</h2>
<p class="d180-md-p">If you’re building alongside this journey, here are a few things worth trying before moving to Day 7.</p>
<hr class="d180-md-hr" />
<h3 id="1-verify-persistence-properly" class="d180-md-h3">1️⃣ Verify Persistence Properly</h3>
<p class="d180-md-p">Restart the app multiple times.</p>
<p class="d180-md-p">Observe:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">Does the entry count increase?</li>
<li class="d180-md-li">Does data survive app restarts?</li>
<li class="d180-md-li">Does it survive a Metro reload?</li>
</ul>
<p class="d180-md-p">Now try:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">Kill the app completely.</li>
<li class="d180-md-li">Reopen it.</li>
</ul>
<p class="d180-md-p">You should still see the existing entries count increase with every new run.</p>
<p class="d180-md-p">If data disappears, something is wrong in your Realm setup.</p>
<hr class="d180-md-hr" />
<h3 id="2-break-the-encryption-intentionally" class="d180-md-h3">2️⃣ Break the Encryption Intentionally</h3>
<p class="d180-md-p">Go to <code class="d180-md-inline-code">realm.ts</code>.</p>
<p class="d180-md-p">Temporarily replace:</p>

      <div class="relative my-6 code-block">
        <div class="absolute right-3 top-3 flex items-center gap-2 z-50 pointer-events-auto">
          <span class="d180-code-lang">TSX</span>
          <button data-copy class="copy-btn d180-copy-btn">Copy</button>
        </div>
        <pre class="d180-code-pre line-numbers"><code class="language-tsx">encryptionKey: key
</code></pre>
      </div>
    
<p class="d180-md-p">With:</p>

      <div class="relative my-6 code-block">
        <div class="absolute right-3 top-3 flex items-center gap-2 z-50 pointer-events-auto">
          <span class="d180-code-lang">TSX</span>
          <button data-copy class="copy-btn d180-copy-btn">Copy</button>
        </div>
        <pre class="d180-code-pre line-numbers"><code class="language-tsx">encryptionKey: generateEncryptionKey()
</code></pre>
      </div>
    
<p class="d180-md-p">Now restart the app.</p>
<p class="d180-md-p">It should fail to open the database.</p>
<p class="d180-md-p">That failure is good.</p>
<p class="d180-md-p">It proves encryption is real.</p>
<p class="d180-md-p">Revert the change after testing.</p>
<hr class="d180-md-hr" />
<h3 id="3-add-a-manual-delete-test" class="d180-md-h3">3️⃣ Add a Manual Delete Test</h3>
<p class="d180-md-p">Inside <code class="d180-md-inline-code">App.tsx</code>, after fetching entries:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">Pick the first entry.</li>
<li class="d180-md-li">Call <code class="d180-md-inline-code">repo.delete(id)</code>.</li>
<li class="d180-md-li">Log the count again.</li>
</ul>
<p class="d180-md-p">Observe how repository encapsulates database operations cleanly.</p>
<p class="d180-md-p">You’ll start feeling the value of abstraction.</p>
<hr class="d180-md-hr" />
<h3 id="4-try-calling-realm-directly-then-undo-it" class="d180-md-h3">4️⃣ Try Calling Realm Directly (Then Undo It)</h3>
<p class="d180-md-p">Inside <code class="d180-md-inline-code">App.tsx</code>, import Realm directly and create an entry.</p>
<p class="d180-md-p">Notice:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">How messy it feels.</li>
<li class="d180-md-li">How you’re bypassing architectural rules .</li>
<li class="d180-md-li">How UI now knows too much about storage.</li>
</ul>
<p class="d180-md-p">Delete that code immediately after observing.</p>
<p class="d180-md-p">This exercise makes boundaries real.</p>
<hr class="d180-md-hr" />
<h3 id="5-add-a-console-log-inside-repository" class="d180-md-h3">5️⃣ Add a Console Log Inside Repository</h3>

        <div class="absolute right-3 top-3 flex items-center gap-2 z-50 pointer-events-auto">
          <span class="d180-code-lang">TSX</span>
          <button data-copy class="copy-btn d180-copy-btn">Copy</button>
        </div>
        <pre class="d180-code-pre line-numbers"><code class="language-tsx">console.log('[Repository] Creating entry');

<p class="d180-md-p">Now watch the flow in logs.</p>
<p class="d180-md-p">You’ll begin to see:</p>
<p class="d180-md-p">UI → Repository → Realm</p>
<p class="d180-md-p">Tomorrow:</p>
<p class="d180-md-p">UI → Service → Repository → Realm</p>
<p class="d180-md-p">Layering becomes visible.</p>

<h2 id="-what-this-phase-is-really-about" class="d180-md-h2">🧠 What This Phase Is Really About</h2>
<p class="d180-md-p">This isn’t about CRUD.</p>
<p class="d180-md-p">It’s about discipline.</p>
<p class="d180-md-p">The repository is the warehouse clerk.</p>
<p class="d180-md-p">It doesn’t decide business rules.</p>
<p class="d180-md-p">It doesn’t validate input.</p>
<p class="d180-md-p">It only handles structured data access.</p>
<p class="d180-md-p">And that separation is what protects the system long-term .</p>
