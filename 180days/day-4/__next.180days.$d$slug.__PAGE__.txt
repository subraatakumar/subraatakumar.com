<h2 id="43-encryption-key-strategy" class="text-2xl lg:text-3xl font-black tracking-tight text-slate-900 dark:text-white mt-14 mb-5">4.3 Encryption Key Strategy</h2>
<h3 id="431-requirement" class="text-xl lg:text-2xl font-bold tracking-tight text-slate-900 dark:text-white mt-10 mb-4">4.3.1 Requirement</h3>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">From the product definition , the application must:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Store all data locally</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Encrypt the database</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Generate a unique key per installation</li>
</ul>
<h3 id="432-implementation-goals" class="text-xl lg:text-2xl font-bold tracking-tight text-slate-900 dark:text-white mt-10 mb-4">4.3.2 Implementation Goals</h3>
<ol class="list-decimal pl-7 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5 pl-1">Generate a 64-byte random encryption key on first launch.</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5 pl-1">Store the key securely:
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">iOS → Keychain</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5 pl-1">Android → Keystore</li>
</ul>
</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5 pl-1">Never store the key in AsyncStorage.</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5 pl-1">Never expose the key in production logs.</li>
</ol>

<h3 id="real-world-analogy" class="text-xl lg:text-2xl font-bold tracking-tight text-slate-900 dark:text-white mt-10 mb-4">Real-World Analogy</h3>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">To understand the Repository Layer, imagine a large warehouse that stores thousands of goods. The warehouse is organized with shelves, codes, storage rules, and inventory systems. It is complex inside.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Now imagine the staff working in the office. When they need an item, They do not search shelves, decode storage labels, or manage inventory records.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Instead, they approach the <strong>warehouse clerk</strong>.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The warehouse clerk knows:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Where items are stored</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">How inventory is organized</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">How goods are retrieved</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">How returns are handled</li>
</ul>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The office staff simply say, “Bring item X.”</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The clerk retrieves it and hands it over.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">In this analogy:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">The <strong>warehouse</strong> represents the database.</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">The <strong>warehouse clerk</strong> represents the repository.</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">The <strong>office staff</strong> represent the rest of the application.</li>
</ul>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The repository hides the complexity of storage. It provides a clean and controlled way to access data. The application does not need to know how the database is structured internally — it only communicates with the repository.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">This improves clarity, reduces duplication, and prevents different parts of the application from interacting with storage in inconsistent ways.</p>

<h3 id="repository-pattern-trade-offs-and-justification" class="text-xl lg:text-2xl font-bold tracking-tight text-slate-900 dark:text-white mt-10 mb-4">Repository Pattern: Trade-offs and Justification</h3>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Some developers argue that introducing a repository layer can add unnecessary complexity, especially in small projects. Modern ORMs often already provide abstraction over database operations, making an additional repository seem redundant or even a potential performance overhead. In simple CRUD applications, wrapping an ORM with another layer may result in boilerplate code without delivering meaningful architectural benefits.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">However, in the context of Offline Life Archive, the repository layer is not merely an ORM wrapper — it enforces architectural boundaries. It prevents direct database access from UI components, isolates business logic, supports encryption control, and keeps the system extraction-ready for future evolution. The slight increase in structural code is a deliberate trade-off for long-term maintainability, modularity, and security integrity.</p>

<h2 id="46-service-layer" class="text-2xl lg:text-3xl font-black tracking-tight text-slate-900 dark:text-white mt-14 mb-5">4.6 Service Layer</h2>
<h2 id="why-the-warehouse-clerk-is-not-enough" class="text-2xl lg:text-3xl font-black tracking-tight text-slate-900 dark:text-white mt-14 mb-5">Why the Warehouse Clerk Is Not Enough</h2>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">In the previous section, we saw how the warehouse clerk (Repository) protects access to the warehouse (Database). Office staff do not walk into storage directly — they request items through the clerk. This keeps storage organized and controlled.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">However, imagine the office staff now need more than simple item retrieval.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Suppose they ask:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">“Can we ship 500 units today?”</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">“Is this customer eligible for priority dispatch?”</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">“Do we have enough stock across multiple sections?”</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">“If one item is unavailable, should we cancel the entire order?”</li>
</ul>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The warehouse clerk cannot answer these questions. The clerk only knows how to retrieve items from shelves.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">This is where the <strong>Warehouse Manager</strong> becomes essential.</p>
<hr>
<h2 id="the-role-of-the-warehouse-manager" class="text-2xl lg:text-3xl font-black tracking-tight text-slate-900 dark:text-white mt-14 mb-5">The Role of the Warehouse Manager</h2>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The warehouse manager does not personally fetch goods from shelves. Instead, the manager:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Applies company rules</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Checks policies and limits</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Validates requests</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Coordinates multiple item retrievals</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Decides whether an operation should proceed</li>
</ul>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Only after making those decisions does the manager instruct the clerk to retrieve items.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">In our architecture:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5"><strong>Warehouse</strong> → Database</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5"><strong>Warehouse Clerk</strong> → Repository</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5"><strong>Warehouse Manager</strong> → Service Layer</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5"><strong>Office Staff</strong> → UI</li>
</ul>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The manager ensures business rules are followed.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The clerk ensures data is retrieved correctly.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Without a manager, staff would rely only on the clerk. Storage would remain organized, but business logic would become inconsistent and scattered.</p>
<hr>
<h2 id="architectural-insight" class="text-2xl lg:text-3xl font-black tracking-tight text-slate-900 dark:text-white mt-14 mb-5">Architectural Insight</h2>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Repository protects storage access.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Service protects business rules.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">You need both for a scalable system.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">This layered separation aligns directly with the architectural philosophy defined earlier — keeping business logic isolated and preventing cross-contamination between layers.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">That is why the Service Layer exists — even when the Repository Layer is already in place.</p>
<h3 id="461-purpose" class="text-xl lg:text-2xl font-bold tracking-tight text-slate-900 dark:text-white mt-10 mb-4">4.6.1 Purpose</h3>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The service layer contains business logic.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">It does <strong>not</strong>:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Render UI</li>

<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">It does:</p>

<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Transform models</li>

<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">We are preparing structure before adding complexity.</p>
<hr>
<h2 id="406-why-this-step-matters" class="text-2xl lg:text-3xl font-black tracking-tight text-slate-900 dark:text-white mt-14 mb-5">4.0.6 Why This Step Matters</h2>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Many developers rush into feature implementation without:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Verifying environment stability</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Cleaning default scaffolding</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Preparing folder structure</li>
</ul>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">We do the opposite.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">We:</p>
<ol class="list-decimal pl-7 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5 pl-1">Ensure environment works.</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5 pl-1">Establish clean project boundaries.</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5 pl-1">Then begin infrastructure implementation.</li>
</ol>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Architecture without foundation is theory.</p>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Architecture inside a running project is engineering.</p>
<hr>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">Once this setup is complete, we are ready to implement:</p>
<ul class="list-disc pl-6 space-y-1 my-4">
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Encrypted Realm initialization</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Repository layer</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">Service layer</li>
<li class="text-[17px] leading-relaxed text-slate-700 dark:text-slate-300 my-1.5">End-to-end data flow</li>
</ul>
<p class="text-[17px] leading-[1.85] text-slate-700 dark:text-slate-300 my-5">The foundation is now ready.</p>
