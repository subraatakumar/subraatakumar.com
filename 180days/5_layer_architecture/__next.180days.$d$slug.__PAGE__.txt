a:Te8d,<h1 id="the-5-layer-architecture-model" class="d180-md-h1">The 5-Layer Architecture Model</h1>
<p class="d180-md-p">We are using a strict layered model:</p>

      <div class="relative my-6 code-block">
        <div class="absolute right-3 top-3 flex items-center gap-2 z-50 pointer-events-auto">
          
          <button data-copy class="copy-btn d180-copy-btn">Copy</button>
        </div>
        <pre class="d180-code-pre line-numbers"><code>App Layer
Module Layer
Core Layer
Shared Layer
Data Layer
</code></pre>
      </div>
    
<p class="d180-md-p">Let’s break that down.</p>
<h3 id="app-layer" class="d180-md-h3">App Layer</h3>
<p class="d180-md-p">Navigation and global orchestration only.</p>
<h3 id="module-layer" class="d180-md-h3">Module Layer</h3>
<p class="d180-md-p">Feature-level business logic (archive, tagging, sharing, settings).</p>
<h3 id="core-layer" class="d180-md-h3">Core Layer</h3>
<p class="d180-md-p">Infrastructure:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">Encryption</li>
<li class="d180-md-li">Secure storage</li>
<li class="d180-md-li">Filesystem</li>
<li class="d180-md-li">Config</li>
<li class="d180-md-li">Devtools</li>
</ul>
<h3 id="shared-layer" class="d180-md-h3">Shared Layer</h3>
<p class="d180-md-p">Reusable, presentational UI components only.</p>
<h3 id="data-layer" class="d180-md-h3">Data Layer</h3>
<p class="d180-md-p">Encrypted Realm database.</p>
<p class="d180-md-p">Each layer has boundaries.</p>
<p class="d180-md-p">No cross-contamination.</p>
<p class="d180-md-p">If layers bleed into each other, refactoring becomes inevitable.</p>
<hr class="d180-md-hr" />
<h1 id="folder-structure-production-ready-from-day-one" class="d180-md-h1">Folder Structure (Production-Ready From Day One)</h1>
<p class="d180-md-p">We created a structure that supports growth:</p>

      <div class="relative my-6 code-block">
        <div class="absolute right-3 top-3 flex items-center gap-2 z-50 pointer-events-auto">
          
          <button data-copy class="copy-btn d180-copy-btn">Copy</button>
        </div>
        <pre class="d180-code-pre line-numbers"><code>src/
  app/
  core/
  modules/
  shared/
</code></pre>
      </div>
    
<p class="d180-md-p">This is not over-engineering.</p>
<p class="d180-md-p">This is prevention.</p>
<p class="d180-md-p">Most MVPs collapse because everything lives inside <code class="d180-md-inline-code">screens/</code>.</p>
<p class="d180-md-p">We are separating:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">Infrastructure from features</li>
<li class="d180-md-li">Business logic from UI</li>
<li class="d180-md-li">Storage from services</li>
<li class="d180-md-li">Reusable systems from product modules</li>
</ul>
<hr class="d180-md-hr" />
<h1 id="strict-architectural-rules-non-negotiable" class="d180-md-h1">Strict Architectural Rules (Non-Negotiable)</h1>
<p class="d180-md-p">From today onward:</p>
<ul class="d180-md-ul">
<li class="d180-md-li">No business logic inside UI components.</li>
<li class="d180-md-li">No direct Realm calls inside screens.</li>
<li class="d180-md-li">Repository layer handles database access.</li>
<li class="d180-md-li">Services handle business logic.</li>
<li class="d180-md-li">Encryption lives only inside <code class="d180-md-inline-code">core/security</code>.</li>
<li class="d180-md-li">File handling lives only inside <code class="d180-md-inline-code">core/fileSystem</code>.</li>
<li class="d180-md-li">Modules never depend directly on other modules.</li>
<li class="d180-md-li">No hardcoded theme values in UI.</li>
</ul>
<p class="d180-md-p">These rules are boring.</p>
<p class="d180-md-p">And that’s the point.</p>
<p class="d180-md-p">Boring architecture scales.</p>
