<h3 id="a-quick-detour-absolute-import-aliases" class="d180-md-h3">A Quick Detour: Absolute Import Aliases</h3>
<p class="d180-md-p">As our project grows, you'll notice import paths getting longer and more fragile.</p>
<p class="d180-md-p"><code class="d180-md-inline-code">import { useArchiveEntries } from '../hooks/useArchiveEntries';</code></p>
<p class="d180-md-p">This is called a <strong>relative path</strong>. If we move this screen to a different folder, the <code class="d180-md-inline-code">../</code> part breaks. It’s brittle.</p>
<p class="d180-md-p">The solution is to use <strong>absolute import aliases</strong>. We can tell our project that the special character <code class="d180-md-inline-code">@/</code> always means "start from the <code class="d180-md-inline-code">src/</code> directory."</p>
<p class="d180-md-p">This is a two-step process:</p>
<h4 id="1-configure-tsconfigjson" class="d180-md-h4">1. Configure <code class="d180-md-inline-code">tsconfig.json</code></h4>
<p class="d180-md-p">Add the <code class="d180-md-inline-code">baseUrl</code> and <code class="d180-md-inline-code">paths</code> properties to your <code class="d180-md-inline-code">tsconfig.json</code> file. This tells TypeScript how to resolve the <code class="d180-md-inline-code">@/</code> alias.</p>
<p class="d180-md-p"><strong>File: <code class="d180-md-inline-code">tsconfig.json</code></strong></p>

      <div class="relative my-6 code-block">
        <div class="absolute right-3 top-3 flex items-center gap-2 z-50 pointer-events-auto">
          <span class="d180-code-lang">JSON</span>
          <button data-copy class="copy-btn d180-copy-btn">Copy</button>
        </div>
        <pre class="d180-code-pre line-numbers"><code class="language-json">{
  "compilerOptions": {
    // ... existing options
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
  // ... rest of the file
}
</code></pre>
      </div>
    
<h4 id="2-update-babel-for-metro-bundler" class="d180-md-h4">2. Update Babel (for Metro Bundler)</h4>
<p class="d180-md-p">React Native's bundler, Metro, doesn't read <code class="d180-md-inline-code">tsconfig.json</code>. We need to tell it about our alias separately using the <code class="d180-md-inline-code">babel-plugin-module-resolver</code>.</p>
<p class="d180-md-p">First, install it:</p>

      <div class="relative my-6 code-block">
        <div class="absolute right-3 top-3 flex items-center gap-2 z-50 pointer-events-auto">
          <span class="d180-code-lang">BASH</span>
          <button data-copy class="copy-btn d180-copy-btn">Copy</button>
        </div>
        <pre class="d180-code-pre line-numbers"><code class="language-bash">npm install --save-dev babel-plugin-module-resolver
</code></pre>
      </div>
    
<p class="d180-md-p">Next, create or update your <code class="d180-md-inline-code">babel.config.js</code> file in the project root:</p>
<p class="d180-md-p"><strong>File: <code class="d180-md-inline-code">babel.config.js</code></strong></p>

      <div class="relative my-6 code-block">
        <div class="absolute right-3 top-3 flex items-center gap-2 z-50 pointer-events-auto">
          <span class="d180-code-lang">JAVASCRIPT</span>
          <button data-copy class="copy-btn d180-copy-btn">Copy</button>
        </div>
        <pre class="d180-code-pre line-numbers"><code class="language-javascript">module.exports = {
  presets: ['module:metro-react-native-babel-preset'],
  plugins: [
    [
      'module-resolver',
      {
        root: ['./src'],
        extensions: ['.ios.js', '.android.js', '.js', '.ts', '.tsx', '.json'],
        alias: {
          "@": "./src",
        }
      }
    ]
  ]
};
</code></pre>
      </div>
    
<h4 id="the-result-cleaner-stronger-imports" class="d180-md-h4">The Result: Cleaner, Stronger Imports</h4>
<p class="d180-md-p">Now, we can refactor our imports to be much cleaner and more robust.</p>
<p class="d180-md-p"><strong>Before:</strong>
<code class="d180-md-inline-code">import { useArchiveEntries } from '../hooks/useArchiveEntries';</code></p>
<p class="d180-md-p"><strong>After:</strong>
<code class="d180-md-inline-code">import { useArchiveEntries } from '@/hooks/useArchiveEntries';</code></p>
<p class="d180-md-p">No matter where we move our screen file, this import will never break. It's a small change that pays huge dividends in maintainability.</p>
